Redesign

Overview

Through this project, I analyze and identify flaws in an existing inteface and practice the redesign workflow by creating low and high fidelity prototypes for various screen sizes and building a responsive website based on those prototypes.

The Original Interface: Home Page

The first step in the redesign process was to select an interface and analyze its flaws. I selected the home page of a popular roller skating rink located in San Francisco. Below you will find a preview of the interface.

Usability Evaluation

The next step in the redesign process is to find potential flaws in the interface's usability. Below you will find an evaluation of the interface's usability as defined by its learnability, efficiency, and memorability.

Learnability
  • The navbar makes it easy for first-time users to navigate to different pages
  • The body of the site is scattered with unaligned images, videos, and floating links, that make it difficult to locate key information, like address, hours of operation, and contact info
  • Both hyperlinked and standard underlined text are red at times, making it difficult to distinguish one from the other
  • The user must discover which flyers/images on the page are linked to other pages. Not all do!
Efficiency
  • Because this interface lacks any sort of hierarchy or categorization in the page's body, even a user who is familiar with the interface would need to scroll and pick through scattered media, making featured media perusing inefficient.
  • Navigation bar usage would be efficient for experienced users.
Memorability
  • The navbar is easily memorable and will allow returning users to regain proficiency easily
  • Because of the overwhelming amount of content in the body of the home page, users could have a less than easy time relearning how to use the body portion of interface effectively.

Accessibility Evaluation

This interface had several accessibility concerns including a few major concerns. The biggest concern being that the navigation bar was not at all identified by the screenreader, meaning that these options would not be accessible for someone relying on a screenreader. The WAVE report indicated that this interface had no page or region structure. The interface used many tables with no data, included empty links, and did not include any alt text.

Final Responsive Redesign

You can view my final redesign here: https://redesigned-church.herokuapp.com/

After running a screenreader and WAVE analysis on my final product, I managed to fixed some of the major issues with the original interface using solely html tags to define hierarchy. I hope the owners of the original interface will choose to implement these changes into their site so it can be more accessible to their users.
